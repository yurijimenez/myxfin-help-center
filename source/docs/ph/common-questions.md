<div class="faq-item">
    <button class="faq-question">
        <span>Paano gumawa ng Sales Order?</span>
        <svg class="w-5 h-5 transition-transform" fill="none" stroke="currentColor" viewBox="0 0 24 24"><path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M19 9l-7 7-7-7"/></svg>
    </button>
    <div class="faq-answer">
        <p>Pumunta sa <strong>Sales & Delivery > Sales Order</strong>. I-click ang "Create New (F1)", ilagay ang customer at item details, at pindutin ang <strong>CTRL + S</strong> para i-save.</p>
    </div>
</div>

<div class="faq-item">
    <button class="faq-question">
        <span>Maaari bang i-void ang transaction na na-deliver na?</span>
        <svg class="w-5 h-5 transition-transform" fill="none" stroke="currentColor" viewBox="0 0 24 24"><path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M19 9l-7 7-7-7"/></svg>
    </button>
    <div class="faq-answer">
        <p>Hindi. Ang mga transactions na may active reference sa Delivery Receipt o Sales Invoice ay hindi maaaring i-void. Kailangan munang i-cancel ang mga kaugnay na documents.</p>
    </div>
</div>

<div class="faq-item">
    <button class="faq-question">
        <span>Ano ang pinagkaiba ng "Billed To" at "Delivered To"?</span>
        <svg class="w-5 h-5 transition-transform" fill="none" stroke="currentColor" viewBox="0 0 24 24"><path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M19 9l-7 7-7-7"/></svg>
    </button>
    <div class="faq-answer">
        <p><strong>Billed To:</strong> Ang entity na magbabayad (lumalabas sa Invoice). <br><strong>Delivered To:</strong> Ang pisikal na lokasyon kung saan ihahatid ang items (lumalabas sa Delivery Receipt).</p>
    </div>
</div>

<div class="faq-item">
    <button class="faq-question">
        <span>Paano gumawa ng Batch Approval?</span>
        <svg class="w-5 h-5 transition-transform" fill="none" stroke="currentColor" viewBox="0 0 24 24"><path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M19 9l-7 7-7-7"/></svg>
    </button>
    <div class="faq-answer">
        <p>Sa Sales Order list view, gamitin ang checkboxes para pumili ng maraming "Pending" transactions, pagkatapos ay i-click ang <strong>Green Checkmark</strong> button sa taas.</p>
    </div>
</div>

<div class="faq-item">
    <button class="faq-question">
        <span>Bakit hindi ko ma-edit ang aking transaction?</span>
        <svg class="w-5 h-5 transition-transform" fill="none" stroke="currentColor" viewBox="0 0 24 24"><path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M19 9l-7 7-7-7"/></svg>
    </button>
    <div class="faq-answer">
        <p>Hindi puwedeng i-edit kapag ang transaction ay <strong>Posted</strong> o <strong>Cancelled</strong> na. Tanging "Draft" o "Pending" lamang ang maaaring i-edit.</p>
    </div>
</div>

<div class="faq-item">
    <button class="faq-question">
        <span>Paano maghanap ng specific Item Code?</span>
        <svg class="w-5 h-5 transition-transform" fill="none" stroke="currentColor" viewBox="0 0 24 24"><path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M19 9l-7 7-7-7"/></svg>
    </button>
    <div class="faq-answer">
        <p>Sa loob ng transaction form, i-click ang "Details" field at magsimulang mag-type ng Item Code o Description. Automatic na magfi-filter ang system.</p>
    </div>
</div>

<div class="faq-item">
    <button class="faq-question">
        <span>Saan ko makikita ang Account Statements?</span>
        <svg class="w-5 h-5 transition-transform" fill="none" stroke="currentColor" viewBox="0 0 24 24"><path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M19 9l-7 7-7-7"/></svg>
    </button>
    <div class="faq-answer">
        <p>Pumunta sa <strong>Finance</strong> module at piliin ang <strong>Statement of Account (SOA)</strong>. Maaari kang mag-filter gamit ang date range at Customer ID.</p>
    </div>
</div>

<div class="faq-item">
    <button class="faq-question">
        <span>Ano ang ginagawa ng "Undo Edit" button?</span>
        <svg class="w-5 h-5 transition-transform" fill="none" stroke="currentColor" viewBox="0 0 24 24"><path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M19 9l-7 7-7-7"/></svg>
    </button>
    <div class="faq-answer">
        <p>Ang <strong>Red Undo Button</strong> ay nagtatanggal ng lahat ng unsaved changes at ibinabalik ang document sa huling saved state nito.</p>
    </div>
</div>

<div class="faq-item">
    <button class="faq-question">
        <span>Paano magdagdag ng items mula sa Quote?</span>
        <svg class="w-5 h-5 transition-transform" fill="none" stroke="currentColor" viewBox="0 0 24 24"><path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M19 9l-7 7-7-7"/></svg>
    </button>
    <div class="faq-answer">
        <p>Sa loob ng Sales Order form, i-click ang <strong>Light Blue Button</strong> upang magbukas ng modal kung saan puwedeng pumili ng existing Quotation.</p>
    </div>
</div>

<div class="faq-item">
    <button class="faq-question">
        <span>May shortcut ba para sa pag-save?</span>

</div>

<div class="faq-item">
    <button class="faq-question">
        <span>Paano palitan ang password?</span>
        <svg class="w-5 h-5 transition-transform" fill="none" stroke="currentColor" viewBox="0 0 24 24"><path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M19 9l-7 7-7-7"/></svg>
    </button>
    <div class="faq-answer">
        <p>
            1. I-click ang <strong>User Icon</strong> sa top-right.<br>
            2. Piliin ang <strong>Change Password</strong>.<br>
            3. Ilagay ang bagong password.<br>
            4. I-click ang <strong>Save</strong>.
        </p>
        <p class="mt-2 text-sm text-blue-600 font-semibold">
            Note: Maximum 15 characters lamang ang password.
        </p>
    </div>
</div>

    <button class="faq-question">
        <span>Paano makita ang Access Log history?</span>
        <svg class="w-5 h-5 transition-transform" fill="none" stroke="currentColor" viewBox="0 0 24 24"><path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M19 9l-7 7-7-7"/></svg>

    <div class="faq-answer">

    </div>
</div>

<div class="faq-item">
    <button class="faq-question">
        <span>Ano ang pagkakaiba ng Void at Cancel?</span>
        <svg class="w-5 h-5 transition-transform" fill="none" stroke="currentColor" viewBox="0 0 24 24"><path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M19 9l-7 7-7-7"/></svg>
    </button>
    <div class="faq-answer">
        <div class="space-y-3">
            <p>
                <strong>Void:</strong> Para sa maling transactions. Tinatanggal ang epekto nito pero may record pa rin.
            </p>
            <p>
                <strong>Cancel:</strong> Para sa valid transactions na hindi na itutuloy.
            </p>
            <div class="p-3 bg-amber-50 border-l-4 border-amber-400 text-sm">
                <strong>Important:</strong> Hindi maaaring mag-<strong>Void</strong> kung may reference na (hal. Delivery Receipt). Kailangan munang i-cancel ang reference.
            </div>
        </div>
    </div>
</div>
